# WSL Development Setup Guide

## Problem: Permission Issues with Windows Filesystem

When developing in WSL with files stored on the Windows filesystem (`/mnt/c/`), you may encounter permission errors like:
```
EACCES: permission denied, rename '/mnt/c/.../node_modules/.pnpm/postcss@8.4.31/node_modules/postcss_tmp_771' -> '/mnt/c/.../node_modules/.pnpm/postcss@8.4.31/node_modules/postcss'
```

## Recommended Solutions

### 1. Use Linux Filesystem (Recommended)
Store your development projects in the Linux filesystem instead of Windows:

```bash
# Create a dedicated development directory
mkdir -p ~/dev
cd ~/dev

# Clone or move your project here
git clone <your-repo-url> my-project
cd my-project

# Install dependencies (should work without permission issues)
pnpm install
```

### 2. WSL Configuration for Windows Filesystem
If you must use Windows filesystem, add these configurations:

#### Update `.wslconfig` in Windows
Create or edit `C:\Users\<username>\.wslconfig`:
```ini
[wsl2]
# Enable metadata for better file system performance
metadata=true
# Enable case sensitivity
caseSensitive=true
# Enable interop for better Windows integration
interopEnabled=true
```

#### Update WSL configuration
```bash
# Edit WSL config
sudo nano /etc/wsl.conf

# Add these lines:
[automount]
enabled = true
root = /mnt/
options = "metadata,umask=22,fmask=11"
```

### 3. Alternative Package Managers
If pnpm continues to have issues, try these alternatives:

#### Using npm
```bash
npm install
```

#### Using yarn
```bash
yarn install
```

#### Using pnpm with different settings
```bash
# Clear pnpm cache
pnpm store prune

# Install with different settings
pnpm install --no-optional --ignore-scripts
```

### 4. File Permission Fixes
If you encounter permission issues:

```bash
# Fix ownership of the entire project
sudo chown -R $(whoami):$(whoami) /mnt/c/Users/TimNjeru/CODING/turborepo

# Fix permissions
chmod -R 755 /mnt/c/Users/TimNjeru/CODING/turborepo

# For node_modules specifically
sudo chown -R $(whoami):$(whoami) /mnt/c/Users/TimNjeru/CODING/turborepo/node_modules
```

## Best Practices

1. **Always use Linux filesystem for development** - This prevents most permission issues
2. **Keep node_modules in .gitignore** - Never commit node_modules
3. **Use consistent package managers** - Stick to one package manager per project
4. **Regular cleanup** - Periodically clean node_modules and reinstall
5. **Use WSL2** - WSL2 has better file system performance than WSL1

## Troubleshooting Commands

```bash
# Check current user and permissions
whoami
id

# Check file permissions
ls -la /mnt/c/Users/TimNjeru/CODING/turborepo

# Check WSL version
wsl --list --verbose

# Restart WSL if needed
wsl --shutdown
# Then restart your terminal
```
